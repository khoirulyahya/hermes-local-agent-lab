# Pelajaran Penting

Beberapa pelajaran paling penting dari eksperimen ini:

## 1. Jangan percaya klaim model tentang dirinya sendiri

Model bisa salah menyebut identitas. Cek konfigurasi dan log, bukan hanya jawaban model.

## 2. Fallback jangan terlalu panjang

Fallback yang terlalu panjang justru bikin lambat dan susah dibaca.

## 3. Gateway harus pakai allowlist

Telegram dan WhatsApp gateway memberi akses ke agent lokal. Itu harus dibatasi.

## 4. MCP sangat berguna, tapi harus dikontrol

MCP Trello bisa membaca dan mengubah card. Karena itu perlu aturan approval.

## 5. Evidence harus jujur

Kalau sesuatu baru eksperimen, tulis eksperimen. Kalau sudah terbukti, tulis verified. Jangan overclaim.

## 6. Dokumentasi publik harus disanitasi

Raw export, token, chat ID, path lokal, link internal, dan detail project private tidak boleh masuk repo publik.
