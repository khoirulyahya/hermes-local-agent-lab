# Alur Aman Coding dengan Agent

Bagian ini penting karena agent yang bisa coding juga bisa berbahaya kalau terlalu bebas.

Maka alurnya dibuat bertahap.

## Alur sederhana

1. Agent baca card Trello.
2. Agent merangkum tugas.
3. Saya memilih card.
4. Agent meminta `approve card`.
5. Agent membuat rencana.
6. Saya memberi `approve plan`.
7. Agent baru boleh edit.
8. Agent menunjukkan diff.
9. Saya memberi `approve commit`.
10. Agent commit.
11. Saya memberi `approve push`.
12. Agent push.
13. Agent update Trello.

## Kenapa tidak langsung otomatis?

Karena tujuan utamanya bukan agent yang bebas melakukan apa pun. Tujuan utamanya adalah agent yang membantu, tapi tetap bisa diawasi.

Itu bedanya automation yang aman dengan automation yang asal jalan.
